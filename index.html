BeatBuddy: Technical Insights and Key Features

Dive Deeper into Our Technical Approach

Learn about the sophisticated technology and algorithms that power BeatBuddy. Our system not only suggests new music but also understands your musical preferences in depth. For a detailed view into our technical infrastructure and methods, check out our Technical Details.

Explore the Unique Features of BeatBuddy:

  • Personalized Recommendations: Tailored song suggestions based on your individual music taste. This feature utilizes advanced techniques such as Variational Matrix Factorization to personalize your music discovery experience.
  • Search by Lyrics: Easily find songs by typing in lyrics or phrases from the music you remember. Our text search leverages BM25 for initial retrieval, followed by COLBERT for reranking, ensuring high relevance and precision.
  • Content-Based Recommendations: Discover songs with similar vibes using word2vec embeddings and numerical features that analyze the musical attributes of tracks.
  • Audio Clip Search: Identify songs from any audio clip using state-of-the-art audio fingerprinting technology. We use spectrograms and local peak analysis to create hashes that efficiently match your clips to our database.

How to Run BeatBuddy Locally

Interested in running BeatBuddy on your own machine? Follow these step-by-step instructions to set up the environment and get everything running locally:

  • Clone the repository: git clone https://github.com/tamu-edu-students/csce-670-beatbuddy.git
  • Install dependencies: npm install for Node.js environments and pip install -r requirements.txt for Python environments.
  • Run the frontend server: Navigate to the frontend directory with cd web/frontend and start the server with npm start.
  • Run the backend server: use python app.py
  • Access the dataset for lyrics, mp3 files, and other necessary data at this link. The dataset was entirely prepared by our team, utilizing the Genius API to acquire song lyrics and a Pytube for downloading MP3 files (for educational purposes only) to generate spectrograms. Numerical features are obtained from the Spotify API and a dataset from Kaggle. To simulate user behavior, we assumed preferences for certain genres, assigning higher ratings to liked genres and lower to disliked ones, thus trying to realistically mimic user interactions.
